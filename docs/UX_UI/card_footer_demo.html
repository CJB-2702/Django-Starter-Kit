How it works

  • Button border-radius is always 0. The buttons are geometrically square. overflow:hidden on the card is what rounds them visually at the corners.
  • The card border draws on top. Clipping happens first (overflow:hidden), then the card's own border traces around the already-clipped shape. That's why the border follows the curve seamlessly — there's no separate "button border" poking out.
  • The gap in Variant A happens because without overflow:hidden, the rectangular button background extends past the card's curve. The border-radius on the card is purely cosmetic at that point — it clips nothing.
  • For the project today (Variant C), all radii are 0, so this is moot. If you ever introduce card rounding, add overflow:hidden to .card and the footer buttons behave correctly with zero extra work.
