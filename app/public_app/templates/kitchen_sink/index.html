HTMX Patterns Demo

format= density switcher

Same canonical URL, different format= query value. Never combine density + htmx-* values.

GET /assets?format=condensed GET /assets?format=medium GET /assets?format=large

HTMX fragment format values

GET /assets?format=htmx-search-results GET /assets?format=htmx-focused

OOB success toast pattern

OOB swap pattern: POST → 303 redirect → HTMX fetches full page → hx-select extracts component → OOB swap injects #toast-container success message.

Session draft flow

Multi-step on one URL: persist draft in request.session['event_draft']. Each sub-component POSTs to update session (no DB commit). Full page at same URL always restores state — F5 rule satisfied.
