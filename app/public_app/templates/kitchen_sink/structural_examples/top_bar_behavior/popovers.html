    ◈ How It Works

    Zero JavaScript. Three HTML/CSS primitives handle everything:

    1. popover="auto" — mutual exclusivity for free. When one auto-popover opens, all others close. Light-dismiss (click outside) also works natively.

    2. commandfor / command="toggle-popover" — the HTML Invoker Commands API (MDN). A button wired to commandfor="portal-X" toggles that popover on click with no script.

    3. CSS :has(.portal-dropdown:popover-open) — the parent <li> gets the active highlight via :has() whenever its child popover is in the :popover-open state. No JS class toggling needed.

    <!-- Each tab li owns both its button AND its popover --> <li role="tab"> <button class="portal-tab-btn" commandfor="portal-events" command="toggle-popover"> ↯ Events ▾ </button> <!-- popover="auto" gives mutual exclusivity + light-dismiss --> <div id="portal-events" popover="auto" class="portal-dropdown"> <p class="portal-dropdown-label">Events</p> <div class="portal-dropdown-inner"> <a href="#" class="portal-sublink">All Events</a> ... </div> </div> </li> /* Active tab highlight — pure CSS, no JS */ li:has(.portal-dropdown:popover-open) button.portal-tab-btn { color: #fff; border-bottom-color: var(--bulma-primary); }

    Browser support: commandfor/command requires Chrome 135+, Firefox 135+. Check caniuse before shipping — no graceful degradation without a small JS polyfill.

    Tradeoff: Links must be server-rendered into the HTML — no runtime data injection. The upside is the links are fully SSR'd, indexable, and accessible without script.
