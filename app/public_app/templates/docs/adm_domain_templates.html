Domain templates

A domain template is a named bundle of data domains representing a real-world scope profile. Instead of assigning domains individually, operators can assign a template and get all related domains at once.

What a template is

A domain template is a convenience tool for bulk domain assignment. Examples:

  • Facility 1 Transportation → [transportation_domain, supply_domain]
  • Warehouse Operations → [warehouse_receiving, warehouse_inventory]
  • Cross-site Auditor → [all_facilities, headquarters]

Assigning a template to a user means: "This user should have access to every domain the template bundles together."

Template as a pointer: Templates reference domain rows; they do not copy domain state. When a domain's configuration changes, every template that references it sees the change immediately.

One active template per user (at most)

Each user can have at most one active domain template assignment at a time.

Users whose scope spans multiple unrelated areas use one of:

  • A single template that bundles all required domains, OR
  • Explicit domain assignments (called "manual" assignments) outside any template

A user can mix: "Assign me Facility 1 Transportation template, plus a manual domain for Cross-facility Auditing."

The rebase rule (default behavior)

When a user's template changes (or is first assigned), the default is rebase:

  1. Remove all domains that came from the previous template
  2. Add every domain from the new template
  3. Leave explicit manual assignments alone

This keeps template changes predictable and prevents scope creep.

Alternative: Additive mode (opt-in)

When assigning a template, operators can choose Additive instead of Rebase. In additive mode, the new template's domains are layered on top of any existing assignments rather than replacing the old template's domains.

Domain template changes are auditable

Every add/remove of a domain to/from a template is tracked:

  • Each template item is timestamped with `created_at`, `updated_at`, `created_by`, `updated_by`
  • When an item is removed, the row is soft-deleted (marked `is_active=False`) rather than permanently deleted
  • Historical views show the complete audit trail: what domains a template contained at any given time, who made changes, and when
Important: Domain templates manage scope only. They have zero effect on capabilities. A user with a domain template still cannot perform actions unless they hold the required roles. The two systems stay separate on purpose.
